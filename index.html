Experiment Tools Purpose

LAIT exists to test Generative AI capabilities and explore what it takes to build AI-powered solutions across any environment. The applications probe current LLM behaviour around structuring and interpreting data, while bringing practical awareness to the business process automations that are already within reach.

Architecture Principles

Local Deployment

The web browser acts as a secure, sandboxed environment — no general access to the application is exposed. Users feed the interface with data only: paste from clipboard or load plain text. Standard CORS policies and browser permission checks block any non-text operations by default, keeping sensitive data entirely on the user's machine.

Simple Architecture

JS files act as pure calculation scripts — performing data transformations and complex math operations with no third-party dependencies. The stack is intentionally minimal: no frameworks, no build pipelines, no external libraries. This makes the application trivially deployable as a standalone external site from any static host.

Moveable to Azure

The same simplicity that enables local deployment also enables cloud migration — as an Azure Static Web App, a Copilot Studio workflow, or an Azure AI Foundry workflow. No complex database or backend relations means zero re-architecture: lift, configure, deploy.

Azure Static Web App Copilot Studio Azure AI Foundry

Manual Exploration of Copilot

A workflow built on copying LLM-generated content directly into the application — no API keys, no permission grants, no integrations required. Local text file loading means the entire flow stays within the user's control, making it a frictionless way to explore Copilot and other LLMs without any infrastructure setup.

PRC — Business problem addressed

As a market researcher preparing reports on product changes, I need to manually gather product information using a browser. I need a solution that generates a list of products, evaluates their market situation based on standardised weights, and summarises the current market situation in a report. These activities take too much of my time.

PRC — Why this works?

The LLM has the ability to browse news on the web targeted at the area indicated by the user.

PRC — Usage of LLM

The LLM searches the web according to the prompt instructions, evaluates information on the basis of the prompt criteria, and compiles a data summary together with a business report.

PRC — Most complex features

The LLM searches news according to the instructions and semantically evaluates information on a scale, then generates a business report.

PRC — Possible extensions

Prompt configuration for topics other than products (e.g. legal changes), expanded dashboard and reporting capabilities (summary built from multiple source reports).

TLAD — Business problem addressed

As a business representative or analyst, I want to be able to define the functionalities of a business system. Based on a written description I want to understand and model a form solution with its lifecycle (low-code solutions), visualise the application scope, identify logical gaps, missing buttons, and unconsidered paths. I have a description of the solution and it is time-consuming to transform that description into the real shape of an application (stages, transition paths) and to visualise it.

TLAD — Why this works?

The LLM is able to process written text into the table structure of an application (elements, connections).

TLAD — Usage of LLM

Based on the prompt and the user's input data, the LLM generates a technical table that is interpretable by the application.

TLAD — Most complex features

The LLM semantically processes text and transforms it into standardised technical tables.

TLAD — Possible extensions

A section for defining form fields (identification based on descriptive data), a more advanced drawing configurator (currently basic functionality), and an application version strictly tailored to BPMN.

BDCP — Business problem addressed

As a business data expert I hold data in the form of a table — source data and a result column representing historical records. I would like to build a business model, verify whether my assumptions are confirmed (relationships between data, understanding of data range min/max), and use that model/algorithm to predict new results. The application could guide me step by step on how to prepare the data and create such an algorithm, and catch potential errors in the algorithm design.

BDCP — Why this works?

The application guides the user step by step on how to configure data for preparing the algorithm. The application measures the effectiveness of linear regression and highlights problems. The LLM report provides descriptive information about the strengths and weaknesses of the model.

BDCP — Usage of LLM

The LLM plays an additional supporting role in evaluating the quality of the algorithm (Linear Regression).

BDCP — Most complex features

An advanced algorithm configurator, analysis of algorithm/model quality, and prediction based on historical data.

BDCP — Possible extensions

Extension to two-dimensional data (normal distribution), more versions of Linear Regression, and enrichment of data with new reports evaluating model quality.
